Assignments

  • Read this article about memory managers from a game programmer's perspective. (Notice the O(n) notation being used. You don't have to completely understand it at this point.)

  • Make sure that you refresh your knowledge of Doxygen for documenting your source code. An example can be found on this page here.

  • Make sure that you review how to compile projects from the command line with these compilers:

    • GNU C++ 4.8.3 (C++11 support) This is the version installed on the computers here and it is also 64-bit. It implements all of C++11.

    • Clang 3.4 (C++11 support) This is the version installed on the computers here. It is 32-bit only for Windows. Linux and Mac OS X have both 32-bit and 64-bit. It implements all of C++11. The nice thing about Clang is that the command-line options are identical to GNU g++, so you can just change g++ to clang++ when compiling.

      Note: Building a project with Clang under Windows may fail to link properly. The code will compile (if you didn't have any errors), but will fail to link properly. This means that no executable will be created, or if one is created, it may fail to execute. You don't have to worry about that at this time. I will only be checking to see that your code compiles cleanly with Clang. You can use this guide to get it working on Windows, if you'd like.

  • Download the preinstalled programs and put them on your computer. There is also a PDF there that explains what to do: readme-2014-RTIS-image.pdf. You can ignore the stuff about Visual Studio, as it's not required for this class.
